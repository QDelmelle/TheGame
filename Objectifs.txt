Histoire:
    RPG:
        On vire ce qui se trouve dans TOUS les RPGs :
	    * balade sur une carte 2D vue du dessus avec apparition aléatoire d'enemis et phase de combat.
	    * boutiques dispersées dans une map qui vendent la même chose
	    * tour par tour simple style final fantaisy
	    * persos bcp trop forts avec 1000 sorts !=
	    * système de cooldown/tour
	Ce qu'on garde des RPGs:
	    * actions avec le décor
	    * gestin quêtes/persos

Nombre de dimensions : 2D


Outil de développement : C++, openframeworks


MUSTS (fonctionnel) :
	Lancement du jeu
	(1)Menu
	(2)Jeu
	(3)Fin du jeu
	
	(1)Menu
		(1.1) Bouton/touche du clavier "Nouvelle partie" => lance une partie depuis le début
		(1.2) Bouton/touche du clavier "Continuer" 	=> reprend là où on s'est arrêté
		(1.3) Bouton/touche du clavier "Quitter le jeu" => ferme le programme
		(1.4) Bouton/touche du clavier "Key Bindings" 	=> permettre de voir les touches du jeu
	(2)Jeu
		(2.1) La carte 	=> un environnement 2D où se trouve un paysage interactif (plus tard)
				=> un quadrillage où chaque carré est une salle
		(2.2) Joueur 	=> Groupe de personnages
				=> chacun à des actions uniques
				=> interdiction de faire des personnages OP ou trop polyvalents
		(2.3) Personnages enemis => ...
		(2.4) Lieux => Sur la carte, des emplacements où se trouveront des villes, des batiments, des grottes
			(2.4.1) Salles => 2D continu avec des ennemis sur certaines 
			coord(x,y), des obstacles/objets sur d'autres
		(2.5) Combat 	=> Dans les salles
				=> phase 1: placement des personnages aux endroits autorisés
    				=> phase 2: Ordres aux persos
				=> importance de la stratégie (placement, où les attaques sont ciblées)
				=> système de stamina limitée/perso*tour, coût != pour chaque action/mouvement. Peut cumuler les actions en 1 tour si la stamina le permet.
				=> Complexité --
				// exemple boxeur : aller tout droit et tabasser le premier type ds son chemin
				// exemple un sniper : donner une direction et il tire une balle qui one shot ...
			(2.5.1) Tours par tours => un moment pour réflechir au prochain move
		(2.6) Menu pause
			(2.6.1) Bouton/touche du clavier "Sauvegarder"
			(2.6.2) Bouton/touche du clavier "Reprendre"
			(2.6.3) Bouton/touche du clavier "Quitter" => retour au (1)Menu
		(2.7) Déplacement => pour avancer dans une salle faudrait remplir un objectif
			(2.7.1) Objectifs pour déplacement 	=> tuer tous les ennemis
								=> dans d'autres casser un truc
								=> trouver un objet
								=> plusieurs objectifs qui ouvrent certaines portes
			
	(3)Fin du jeu
		(3.1) Ecran spécial => ...
		(3.2) Retour au (1)Menu
